# PLAIN_ENGLISH_GUIDE.md

Use this whenever giving Wayne updates.

## Rule
Always explain technical terms in plain English first, then optionally add technical name in brackets.

## Simple translations
- **Sprint** = a short focused work block to finish a small chunk.
- **NS-001 / NS-002** = specific task IDs in the project checklist.
- **Execution chain** = the order tasks should be done.
- **Ship-check gate** = a quality safety checklist we run before moving forward.
- **Checkpoint** = a pause to verify everything is still good.
- **PR (Pull Request)** = a request to merge code changes into the main project.
- **Merge** = combine approved changes into the main version.
- **Main branch** = the current official version of the project.
- **CI** = automatic tests/checks run by GitHub.
- **Blocker** = something stopping progress.
- **No blocker** = nothing is stopping progress.

## Message format for Wayne
Use this exact structure for project updates:
1) **What this means in plain English**
2) **What I did**
3) **What happens next**
4) **Do you need to do anything? (Yes/No + one line)**

## Writing style constraints
- Max 5 bullets.
- Short sentences.
- No jargon without explanation.
- If using IDs like NS-001, explain the task in words on same line.
